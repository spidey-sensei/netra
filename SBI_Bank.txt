## Critical: SBI Balance Rules

- Balance format     : SBI shows balance as "AMOUNT CR" or "AMOUNT DR"
                       CR = Credit balance (positive, money in account)
                       DR = Debit balance (negative, account is overdrawn)
                       Always include CR/DR when reporting balance.

- Opening Balance    : SBI does NOT have an "Opening Balance" labeled row.
                       Calculate it from the first transaction:
                       If first transaction is a Credit → Opening Balance = First row balance - Credit amount
                       If first transaction is a Debit  → Opening Balance = First row balance + Debit amount

- Closing Balance    : The Balance value in the VERY LAST transaction row on the last
                       non-empty page. Do NOT use "Available Balance" from page 1 header
                       as the closing balance — they are different.

- Available Balance  : This is the real-time balance at the time the statement was generated.
                       It is NOT the closing balance of the statement period.

- Last page warning  : SBI statements often have a blank last page with only a disclaimer.
                       Always ignore blank or disclaimer-only pages.
                       The closing balance is on the last page that has actual transaction rows.

- Debit/Credit       : In the Debit column, amounts show as "X.XX DR"
                       In the Credit column, amounts show as "X.XX CR"
                       Strip the DR/CR suffix before doing any arithmetic.


## SBI Transaction Table Structure

Each transaction row has these columns:
- Txn Date      : Transaction date (DD-MM-YY)
- Value Date    : Value date (DD-MM-YY)
- Description   : Narration / description of transaction
- Ref/Cheque No.: Reference or cheque number (usually "000000" for UPI/NEFT)
- Debit         : Amount debited with DR suffix (outgoing)
- Credit        : Amount credited with CR suffix (incoming)
- Balance       : Running balance after transaction with CR or DR suffix

Note: SBI does NOT have a separate Opening Balance row or Statement Summary section.


## How to Categorize SBI Transactions

Use these rules to classify each transaction:
- UPI/DR        → UPI Outgoing Payment (debit)
- UPI/CR        → UPI Incoming Receipt (credit)
- NEFT*         → NEFT Transfer (incoming or outgoing based on Debit/Credit column)
- ACHDr         → Auto-Debit (SIP, loan EMI, insurance)
- ACHCr         → Auto-Credit (dividend, salary, refund)
- IMPS / INB IMPS → Instant Payment
- INB           → Internet Banking transaction
- CHEQUE TRANSFER → Cheque payment
- ATMCard AMC   → ATM card annual maintenance charge
- CASH Withdrawn → Cash withdrawal
- CASH Deposited → Cash deposit
- INTEREST CREDIT / ITDTAX → Interest earned or tax refund
- COMM          → Bank charges / commission
- NIPPON I NACH FAIL → ECS/NACH bounce penalty
- YONOABDS      → SBI YONO app bill payment (mutual fund SIP etc.)

## Critical Calculation Rules

- Opening Balance    : NO labeled row exists. Calculate as:
                       First transaction is Credit → Opening = First Balance - Credit amount
                       First transaction is Debit  → Opening = First Balance + Debit amount

- Closing Balance    : Balance column value of the VERY LAST transaction row
                       on the last non-empty page. Never use Available Balance for this.

- Total Debits       : Sum ALL values from the Debit column across ALL pages.
                       Strip "DR" suffix before summing. Do not stop early.

- Total Credits      : Sum ALL values from the Credit column across ALL pages.
                       Strip "CR" suffix before summing. Do not stop early.

- Net Change         : Closing Balance minus Opening Balance.
                       If result is negative, account had more outflow than inflow.
